# Documentation Generation

Create thorough, useful documentation following best practices.

## Phase 1: Scope Definition
Identify documentation type needed:
- API documentation
- Code documentation (inline)
- Architecture documentation
- User guide / README
- Contributing guide
- Deployment guide

## Phase 2: Content Analysis
1. **Understand the code**
   - Purpose and functionality
   - Key components and interactions
   - Public interfaces
   - Configuration options

2. **Identify audience**
   - End users
   - API consumers
   - Contributors
   - Operators/DevOps

## Phase 3: Documentation Structure

### For APIs:
- Overview and purpose
- Authentication
- Endpoints with:
  - Method and path
  - Parameters
  - Request examples
  - Response examples
  - Error codes
- Rate limiting
- Versioning

### For Code:
- Module/class purpose
- Public API documentation
- Usage examples
- Edge cases and gotchas
- Performance considerations

### For Architecture:
- System overview
- Component diagram
- Data flow
- Technology choices (and why)
- Deployment architecture
- Scalability considerations

### For READMEs:
- What is it?
- Why use it?
- Quick start
- Installation
- Basic usage
- Configuration
- Examples
- Contributing
- License

## Phase 4: Quality Checklist
- ✓ Clear and concise
- ✓ Code examples work
- ✓ Covers common use cases
- ✓ Addresses error scenarios
- ✓ Up-to-date with current code
- ✓ Proper formatting
- ✓ Linked resources
- ✓ Searchable/navigable

## Deliverable
Complete, well-structured documentation ready for users/contributors.
